💾 Sichere Backups

Datensicherung und Recovery-Strategien für Aktivist*innen.

Warum Backups kritisch sind

Aktivist*innen sind besonderen Risiken ausgesetzt: Gerätebeschlagnahmung, Ransomware-Angriffe, physischer Diebstahl oder schlicht Hardware-Defekte. Ohne Backup verlierst du nicht nur Daten, sondern potenziell Jahre an Arbeit.

⚠ Realität: Laut Studien haben 30% der Menschen noch nie ein Backup gemacht. Von denen, die es tun, testen nur 10% die Wiederherstellung. Ein ungetestetes Backup ist kein Backup.

📈 Die 3-2-1-Regel

Der Goldstandard für Datensicherung:

3 Kopien

Mindestens drei Kopien deiner wichtigen Daten.

  • Original auf dem Geraet
  • Erste Backup-Kopie
  • Zweite Backup-Kopie

2 Medientypen

Speichere auf verschiedenen Medien.

  • Interne SSD/HDD
  • Externe Festplatte
  • USB-Stick
  • NAS

1 Offsite

Eine Kopie außer Haus.

  • Bei Vertrauensperson
  • Bankschliessfach
  • Verschluesselte Cloud

🔒 Verschlüsselungs-Tools

VeraCrypt

Für: Externe Laufwerke, Container

  • Open Source, auditiert
  • Versteckte Volumes möglich
  • Plausible Deniability
  • Cross-Platform

veracrypt.fr

Cryptomator

Für: Cloud-Verschlüsselung

  • Open Source
  • Einzelne Dateien verschlüsselt
  • Für Dropbox, Google Drive, etc.
  • Einfache Bedienung

cryptomator.org

💡 Unterschied: VeraCrypt erstellt einen großen Container (wie eine virtuelle Festplatte). Cryptomator verschlüsselt jede Datei einzeln – besser für Cloud-Sync, da nur geänderte Dateien hochgeladen werden.

💻 Backup-Software

Linux

  • Borg Backup – Deduplizierung, Verschlüsselung, sehr effizient
  • restic – Ähnlich wie Borg, einfacher zu nutzen
  • Deja Dup – GUI für duplicity, GNOME-Integration

Windows

  • Duplicati – Open Source, Cloud-Support
  • Kopia – Modern, schnell, verschlüsselt
  • Veeam Agent – Kostenlos für Privatnutzer

macOS

  • Time Machine – Eingebaut, einfach (aber: nicht verschlüsselt auf externen Laufwerken ohne FileVault)
  • Arq – Verschluesselt, Cloud-Support

⚠ Cloud-Backup: Ja oder Nein?

Cloud-Speicher ist praktisch, aber für Aktivist*innen problematisch:

Risiken

  • US-Anbieter unterliegen CLOUD Act
  • Account-Sperrung möglich
  • Metadaten-Analyse durch Anbieter
  • Abhängigkeit von Drittanbietern

Wenn Cloud, dann...

  • Immer lokal verschluesseln (Cryptomator)
  • EU-Anbieter bevorzugen
  • Tresorit, Filen, Proton Drive
  • Eigene Nextcloud-Instanz

📦 Was sichern?

  • Dokumente: Alle persönlichen Dokumente, Scans, Vertraege
  • Passwörter: KeePassXC-Datenbank (oder Bitwarden-Export)
  • 2FA-Codes: Aegis-Export (verschlüsselt!), Recovery-Codes
  • PGP-Keys: Privater Schlüssel sicher exportiert
  • Fotos/Videos: Besonders von Aktionen (Beweismaterial)
  • Kontakte: Export aus Signal, E-Mail-Kontakte
  • Konfiguration: SSH-Keys, VPN-Profile, wichtige Config-Dateien
⚠ Nicht sichern: Temporaere Dateien, Browser-Cache, Downloads die neu heruntergeladen werden können. Halte Backups schlank.

🛠 Recovery testen

Ein Backup ist nur so gut wie der letzte erfolgreiche Test.

  • Regelmäßig testen: Mindestens alle 6 Monate eine vollständige Wiederherstellung
  • Verschiedene Dateien: Nicht nur prüfen, ob das Backup existiert – tatsaechlich Dateien öffnen
  • Auf anderem Geraet: Teste Recovery auf einem anderen Computer
  • Passwort-Check: Kannst du das Verschlüsselungs-Passwort noch? Ist es dokumentiert?

🕵 Backup bei Beschlagnahmung

Besondere Überlegungen für Aktivist*innen:

  • Offsite-Backup essenziell: Wenn Polizei alle Geräte mitnimmt, brauchst du ein Backup außer Haus
  • Plausible Deniability: VeraCrypt Hidden Volumes für sensitive Daten
  • Trusted Person: Eine Vertrauensperson sollte wissen, wo das Backup ist und wie darauf zugegriffen wird
  • Rechtliche Dokumentation: Backup aller Schriftverkehre mit Anwält*innen, Behörden
💡 Dead Man's Switch: Einige Aktivist*innen nutzen Timer-basierte Systeme die automatisch Daten veröffentlichen, wenn sie sich nicht regelmäßig melden. Das ist komplex und riskant – nur für spezielle Situationen.

📅 Backup-Plan erstellen

Was Wohin Wie oft Verschlüsselung
Arbeitsdokumente Externe SSD + Cloud Täglich Cryptomator
Passwörter/Keys USB-Stick Offsite Bei Änderung VeraCrypt
Fotos/Videos Externe HDD Wöchentlich VeraCrypt
System-Image Externe HDD Monatlich VeraCrypt
💡 Zusammenfassung: Folge der 3-2-1-Regel, verschluessele alles, teste regelmäßig und halte mindestens ein Backup außer Haus. Ein Backup das du nicht testen kannst oder dessen Passwort du vergessen hast, ist nutzlos.
